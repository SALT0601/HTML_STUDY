Form

사용자로부터 어떠한 값을 받을 목적으로 기술
전달될 데이터는 form 태그에 포함되어야 한다.
키와 값이 같이 전달된다.
method에 GET과 POST 방식이 있다.
GET : 256Byte ~ 4000Byte 이내의 크기의 데이터만 전송 가능
주소줄에 붙어서 넘어감
address?key=value&keyvalue...이런식으로 넘어가서 정보를 다 볼 수 있음
그래서 데이터를 입력하는 폼이나 패스워드 입력하는 폼은 POST로 넘긴다
POST : 크기 상관 없음.
body 처음에 붙어서 넘어간다.(내용을 볼 수 없다.)
그래서 데이터를 서버에 전송할때는 POST를 쓰고
꺼내올때나 브라우저 정보 알려줄 때 GET을 씀
이렇게 정보 전송을 위해서 name이 중요하다.
name에 지정해준 변수로 이클립스에서 값 받아오기때문이다.

Size는 필드의 크기의
maxlength는 정의한 필드에 입력할 수 있는 최대 값. 사이즈가 20이라도 맥스렝스가 10이면 10만 입력됨
required는 값이 반드시 입력되어야 한다.
readonly는 읽기전용 사용자가 값을 입력할 수 없음
disable : 사용하지 않음. 사용자가 값을 입력할 수 없음. 지정된 value가 있어도 안넘어감.저장한 action에 값을 전달하지 않는다. 그래서 null뜬다
